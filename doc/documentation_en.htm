Overview

With jSync, you can, periodically, synchronize a directory or a bunch of directories, to maintain identical copies of files, for example to maintain the same state of configurations files for an app that runs on two differents servers.

Features

  • Platform independent
  • Easy to use graphical UI (available in English & French)
  • Synchronize sub-directories
  • Delete remaing files in the target directory
  • Criterias to copy file from source to target :
    • date comparaison on files
    • size comparaison on files
    • always
  • multi-channels (multi-thread)

Installation

  • Windows : You can either start jSync by double-clicking on jSync.exe.
  • Un*x : create a "jSync.sh" shell script with the required paths.
    => java -jar jSync.jar

General

A channel is the process that synchronizes directories. You can have multiple channels within the same intance of jSync. The channel configuration is automacally saved after each modification.

Menus

Files:

  • Start all channels : Switch all existing channels on and start synchronizing.
  • Stop all channels : Swicth all existing channels off
  • Clear log Window : Delete logs comments in the window
  • Preference : Open a dialog to change the look and feel
  • Quit : Exit jSync

Channel :

  • New : Create a new channel
  • Modify : Open a dialog to modify the selected channel properties
  • Start : Switch the selected channel on and start synchronizing
  • Stop : Switch the selected channel off
  • Destroy : Delete the selected channel

Channel Properties

  • Name : Channel name
  • Source Directory : Reference source Directory
  • Target Directory : The Directory where files have to be copied
  • Tempo : Time in minutes between to synchronization
  • Include Subdirectories : If selected, process the sub directories iof the source directory
  • Delete orphean files in target directory : If selected, delete file which are present in the target directory but not in the source directory. You have to select this option to maintain the same stucture between the source and target directories.
  • Copy Files : Various synchronization critï¿½rias.

Status of the channel

  • STOPPED : Channel is stopped. No synchronization
  • STARTING : Channel is starting (usually appends for a very short time)
  • ACTIVE : Channel is on, waiting for the next synchro.
  • RUNNING : Synchronization is running
  • ERROR : Error. Look at the log window for more information

Bugs
